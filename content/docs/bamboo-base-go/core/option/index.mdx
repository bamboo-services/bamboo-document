# xOption — 声明式配置

`xOption` 基于**函数式选项模式**，让业务侧以声明式风格选择框架内置组件（缓存后端、数据库驱动、路由挂载），而非手工编写初始化节点。Runner 在启动时根据 `Config` 自动完成装配。

```go
import xOption "github.com/bamboo-services/bamboo-base-go/major/option"
```

## 设计理念

传统的 `xRegNode.RegNodeList` 要求业务侧编写完整的初始化工厂函数（打开连接、拼装 DSN、注册中间件）。对于框架内置组件（缓存、数据库、路由），xOption 将这部分逻辑内化到框架层，业务侧只需描述「选哪种 + 用什么参数」。

```
注册节点模式                  Option 模式
─────────────                ───────────
xReg.Register(ctx, nodeList)  xOption.Apply(...)
  └── 用户手写 InitXxx         └── WithRedis("localhost:6379")
                                  └── WithMySQL("user:pass@tcp(...)/db")
                                  └── WithRoute(routeFunc)
```

## 核心类型

### Option & Config

```go
// Option 是应用级配置选项的函数签名
type Option func(*Config)

// Config 是应用运行期配置的聚合体
type Config struct {
    // 私有字段，通过 getter 暴露只读视图
}

func (c *Config) Cache()    CacheConfig          // 缓存配置
func (c *Config) Database() xOptionDB.Config     // 数据库配置
func (c *Config) Routes()   []RouteRegistrar     // 路由注册器列表
```

### Apply

将 Option 列表逐个应用到 Config，nil 选项自动跳过。

```go
func Apply(opts ...Option) *Config
```

## 使用示例

### 完整配置

```go title="main.go"
package main

import (
    "context"
    "time"

    xLog "github.com/bamboo-services/bamboo-base-go/common/log"
    xMain "github.com/bamboo-services/bamboo-base-go/major/main"
    xOption "github.com/bamboo-services/bamboo-base-go/major/option"
    xReg "github.com/bamboo-services/bamboo-base-go/major/register"
)

func main() {
    reg := xReg.Register(context.Background(), nil)
    log := xLog.WithName(xLog.NamedMAIN)

    // 声明式配置：选择组件实现
    opts := []xOption.Option{
        // 缓存：使用 Redis
        xOption.WithRedis("localhost:6379",
            xOption.WithRedisPassword("xxx"),
            xOption.WithRedisDB(0),
        ),
        // 数据库：MySQL
        xOption.WithMySQL("user:pass@tcp(localhost:3306)/bamboo?charset=utf8mb4&parseTime=True"),
        // 路由
        xOption.WithRoute(func(ctx context.Context, serve *gin.Engine) {
            serve.GET("/ping", func(c *gin.Context) {
                c.String(200, "pong")
            })
        }),
    }

    // Runner 根据 opts 自动装配 Redis 客户端、GORM DB 等
    xMain.Runner(reg, log, opts)

    // 也支持传入后台协程函数
    // xMain.Runner(reg, log, opts, goroutineFunc)
}
```

### 仅缓存 + Memory 后端

```go
opts := []xOption.Option{
    xOption.WithMemory(
        xOption.WithMemoryDefaultTTL(10*time.Minute),
        xOption.WithMemoryMaxEntries(10000),
    ),
    xOption.WithRoute(func(ctx context.Context, serve *gin.Engine) {
        serve.GET("/ping", func(c *gin.Context) {
            c.JSON(200, gin.H{"message": "pong"})
        })
    }),
}
 
xMain.Runner(reg, log, opts)
```

### 从环境变量装配数据库

```go
opts := []xOption.Option{
    xOption.WithDatabaseFromEnv(), // 读取 DATABASE_DRIVER/DATABASE_HOST/... 等
    xOption.WithRoute(routeFunc),
}

xMain.Runner(reg, log, opts)
```

### 条件构造

nil Option 被自动跳过，可安全用于条件分支：

```go
var opts []xOption.Option

if useRedis {
    opts = append(opts, xOption.WithRedis("localhost:6379"))
} else {
    opts = append(opts, xOption.WithMemory())
}

opts = append(opts, xOption.WithRoute(routeFunc))

xMain.Runner(reg, log, opts)
```

## 与注册节点模式的关系

xOption 是注册节点模式的**补充**，而非替代。两者可以共存：

| 场景 | 推荐方式 |
|------|----------|
| 框架内置组件（缓存、数据库、路由） | xOption |
| 自定义初始化逻辑（第三方 SDK、业务启动） | xRegNode |
| 需要访问前序节点结果 | xRegNode（节点化注入） |

```go
// 混合模式：内置组件用 Option，自定义逻辑用节点
reg := xReg.Register(ctx, []xRegNode.RegNodeList{
    {Key: xCtx.CustomClientKey, Node: initCustomSDK},
})

opts := []xOption.Option{
    xOption.WithRedis("localhost:6379"),
    xOption.WithRoute(routeFunc),
}

xMain.Runner(reg, log, opts)
```
