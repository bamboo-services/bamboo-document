# 异常处理概览

`bamboo-webflux` 提供了一套多层级的响应式异常处理器链，基于 Spring 的 `@RestControllerAdvice` 机制，将各类异常统一转换为标准化的 `Mono<ResponseEntity<BaseResponse<T>>>` 响应。

## 处理器层级

异常处理器按照继承关系与职责划分为以下层级：

```
SystemExceptionHandler (@Order(-1), 最高优先级)
└── PublicExceptionHandler
    └── JavaBaseExceptionHandler
```

| 处理器 | 职责 | 覆盖异常类型 |
|--------|------|-------------|
| `SystemExceptionHandler` | Spring WebFlux 框架级异常 | MethodNotAllowedException、WebExchangeBindException 等 |
| `PublicExceptionHandler` | 业务异常 | BusinessException、UserAuthenticationException 等 |
| `JavaBaseExceptionHandler` | Java 标准库异常 | NullPointerException、IllegalArgumentException 等 |

此外还有两个独立的数据库异常处理器：

| 处理器 | 职责 |
|--------|------|
| `MysqlExceptionHandler` | MySQL 数据库异常 |
| `PostgreSqlExceptionHandler` | PostgreSQL 数据库异常 |

## 响应式返回类型

与 `bamboo-mvc` 中直接返回 `ResponseEntity<BaseResponse<T>>` 不同，`bamboo-webflux` 的所有异常处理方法均返回 `Mono<ResponseEntity<BaseResponse<T>>>` 类型：

```java title="ExceptionHandler.java"
// bamboo-mvc
@ExceptionHandler(BusinessException.class)
public ResponseEntity<BaseResponse<Void>> handleBusinessException(BusinessException e) {
    return ResultUtil.error(e.getErrorCode(), e.getMessage(), null);
}

// [!code highlight:4]
// bamboo-webflux（响应式版本）
@ExceptionHandler(BusinessException.class)
public Mono<ResponseEntity<BaseResponse<Void>>> handleBusinessException(BusinessException e) {
    return ResultUtil.error(e.getErrorCode(), e.getMessage(), null);
}
```

## 优先级机制

`SystemExceptionHandler` 通过 `@Order(-1)` 注解获得最高优先级。当多个异常处理器可以处理同一异常时，高优先级的处理器优先生效。异常处理器的继承关系确保了更具体的处理器优先于更通用的处理器。

## 扩展方式

开发者可以通过继承 `SystemExceptionHandler` 来扩展异常处理能力：

```java title="GlobalExceptionHandler.java"
@RestControllerAdvice
public class GlobalExceptionHandler extends SystemExceptionHandler {

    @ExceptionHandler(CustomException.class)
    public Mono<ResponseEntity<BaseResponse<Void>>> handleCustom(CustomException e) {
        return ResultUtil.error(ErrorCode.SERVER_INTERNAL_ERROR, e.getMessage(), null);
    }
}
```
