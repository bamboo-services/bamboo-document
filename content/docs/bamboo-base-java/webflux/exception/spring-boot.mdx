# SystemExceptionHandler

`SystemExceptionHandler` 是异常处理器链的最顶层，通过 `@Order(-1)` 获得最高优先级，负责处理 Spring WebFlux 框架抛出的各类异常。它继承自 `PublicExceptionHandler`，因此同时具备业务异常和 Java 标准库异常的处理能力。

## 类定义

```java title="SystemExceptionHandler.java"
// [!code highlight:4]
// 最高优先级的异常处理器，处理 Spring WebFlux 框架异常
@RestControllerAdvice
@Order(-1)
public class SystemExceptionHandler extends PublicExceptionHandler {
    // ...
}
```

## 处理的异常类型

### MethodNotAllowedException

当请求使用了不被允许的 HTTP 方法时触发。例如，对仅接受 `GET` 的端点发送 `POST` 请求。

```java title="SystemExceptionHandler.java"
@ExceptionHandler(MethodNotAllowedException.class)
public Mono<ResponseEntity<BaseResponse<Void>>> handleMethodNotAllowed(
        MethodNotAllowedException e) {
    // [!code highlight:2]
    // 返回 405 Method Not Allowed
    return ResultUtil.error(ErrorCode.METHOD_NOT_ALLOWED, e.getMessage(), null);
}
```

### MissingRequestValueException

当请求中缺少必要的参数值时触发。涵盖请求参数、路径变量、请求头等场景。

```java title="SystemExceptionHandler.java"
@ExceptionHandler(MissingRequestValueException.class)
public Mono<ResponseEntity<BaseResponse<Void>>> handleMissingRequestValue(
        MissingRequestValueException e) {
    return ResultUtil.error(ErrorCode.PARAMETER_ERROR, e.getMessage(), null);
}
```

### WebExchangeBindException

当请求体参数绑定与校验失败时触发。这是 WebFlux 环境中 `@Valid` / `@Validated` 注解校验失败的标准异常。

```java title="SystemExceptionHandler.java"
@ExceptionHandler(WebExchangeBindException.class)
public Mono<ResponseEntity<BaseResponse<Void>>> handleWebExchangeBind(
        WebExchangeBindException e) {
    // [!code highlight:2]
    // 提取校验失败的字段与错误信息
    return ResultUtil.error(ErrorCode.PARAMETER_ERROR, e.getMessage(), null);
}
```

### ServerWebInputException

当服务端处理请求输入时发生错误触发。通常由请求体反序列化失败引起。

```java title="SystemExceptionHandler.java"
@ExceptionHandler(ServerWebInputException.class)
public Mono<ResponseEntity<BaseResponse<Void>>> handleServerWebInput(
        ServerWebInputException e) {
    return ResultUtil.error(ErrorCode.PARAMETER_ERROR, e.getMessage(), null);
}
```

### UnsupportedMediaTypeStatusException

当请求的 Content-Type 不被端点支持时触发。例如，端点期望 `application/json` 但接收到 `text/plain`。

```java title="SystemExceptionHandler.java"
@ExceptionHandler(UnsupportedMediaTypeStatusException.class)
public Mono<ResponseEntity<BaseResponse<Void>>> handleUnsupportedMediaType(
        UnsupportedMediaTypeStatusException e) {
    // [!code highlight:2]
    // 返回 415 Unsupported Media Type
    return ResultUtil.error(ErrorCode.UNSUPPORTED_MEDIA_TYPE, e.getMessage(), null);
}
```

### NotAcceptableStatusException

当服务端无法生成客户端可接受的响应格式时触发。通常由 `Accept` 请求头与服务端能力不匹配引起。

```java title="SystemExceptionHandler.java"
@ExceptionHandler(NotAcceptableStatusException.class)
public Mono<ResponseEntity<BaseResponse<Void>>> handleNotAcceptable(
        NotAcceptableStatusException e) {
    return ResultUtil.error(ErrorCode.NOT_ACCEPTABLE, e.getMessage(), null);
}
```

### ResponseStatusException

Spring WebFlux 的通用响应状态异常，当其他具体异常处理器未匹配时作为兜底处理。

```java title="SystemExceptionHandler.java"
@ExceptionHandler(ResponseStatusException.class)
public Mono<ResponseEntity<BaseResponse<Void>>> handleResponseStatus(
        ResponseStatusException e) {
    // [!code highlight:2]
    // 从异常中提取 HTTP 状态码，映射为对应的 ErrorCode
    return ResultUtil.error(ErrorCode.SERVER_INTERNAL_ERROR, e.getMessage(), null);
}
```

## 与 MVC 版本的差异

`SystemExceptionHandler` 在 WebFlux 版本中处理的异常类型与 MVC 版本有所不同，反映了两个框架底层异常体系的差异：

| MVC 版本 | WebFlux 版本 |
|----------|-------------|
| `HttpRequestMethodNotSupportedException` | `MethodNotAllowedException` |
| `MissingServletRequestParameterException` | `MissingRequestValueException` |
| `MethodArgumentNotValidException` | `WebExchangeBindException` |
| `HttpMessageNotReadableException` | `ServerWebInputException` |
| `HttpMediaTypeNotSupportedException` | `UnsupportedMediaTypeStatusException` |
| `HttpMediaTypeNotAcceptableException` | `NotAcceptableStatusException` |
| - | `ResponseStatusException` |
