# PublicExceptionHandler

`PublicExceptionHandler` 继承自 `JavaBaseExceptionHandler`，负责处理框架预定义的业务异常与用户认证异常。这些异常通常由开发者在业务逻辑中主动抛出，用于表达可预期的业务错误。

## 类定义

```java title="PublicExceptionHandler.java"
// [!code highlight:3]
// 继承 JavaBaseExceptionHandler，在其基础上增加业务异常处理
@RestControllerAdvice
public class PublicExceptionHandler extends JavaBaseExceptionHandler {
    // ...
}
```

## 处理的异常类型

### BusinessException

通用业务异常，是最常用的异常类型。开发者可在业务逻辑中根据具体场景抛出。

```java title="PublicExceptionHandler.java"
@ExceptionHandler(BusinessException.class)
public Mono<ResponseEntity<BaseResponse<Void>>> handleBusinessException(
        BusinessException e) {
    // [!code highlight:2]
    // 从 BusinessException 中提取 ErrorCode 与消息，构建响应
    return ResultUtil.error(e.getErrorCode(), e.getMessage(), null);
}
```

**业务代码中的使用：**

```java title="UserService.java"
public Mono<UserDTO> getUserById(Long id) {
    return userRepository.findById(id)
            .switchIfEmpty(Mono.error(
                    // [!code highlight:2]
                    // 抛出业务异常，会被 PublicExceptionHandler 捕获
                    new BusinessException(ErrorCode.RESOURCE_NOT_FOUND, "用户不存在")
            ));
}
```

### UserAuthenticationException

用户认证异常，用于表达认证与授权相关的错误。在 `bamboo-webflux` 中，该异常使用 `ServerWebExchange` 替代 `HttpServletRequest` 来获取请求信息。

```java title="PublicExceptionHandler.java"
@ExceptionHandler(UserAuthenticationException.class)
public Mono<ResponseEntity<BaseResponse<Void>>> handleUserAuthenticationException(
        UserAuthenticationException e) {
    // [!code highlight:2]
    // 记录认证失败的详细信息用于安全审计
    return ResultUtil.error(e.getErrorCode(), e.getMessage(), null);
}
```

#### UserAuthenticationException 构造

在 WebFlux 环境中，`UserAuthenticationException` 接受 `ServerWebExchange` 参数：

```java title="UserAuthenticationException.java"
// [!code highlight:3]
// 构造方法使用 ServerWebExchange 而非 HttpServletRequest
public UserAuthenticationException(ErrorType errorType, ServerWebExchange exchange) {
    // ...
}
```

#### UserInfo 记录类

`UserAuthenticationException` 内部包含 `UserInfo` 记录类，用于记录认证失败时的请求信息：

#### 工厂方法

`UserAuthenticationException` 提供一系列语义化的工厂方法：

```java title="UserAuthenticationException.java"
// [!code highlight:9]
// 各类认证失败场景的工厂方法
UserAuthenticationException.tokenExpired(exchange)       // Token 过期
UserAuthenticationException.permissionDenied(exchange)   // 权限不足
UserAuthenticationException.userNotLogin(exchange)       // 用户未登录
UserAuthenticationException.userNotExist(exchange)       // 用户不存在
UserAuthenticationException.wrongPassword(exchange)      // 密码错误
UserAuthenticationException.userBanned(exchange)         // 用户被封禁
UserAuthenticationException.loginWrong(exchange)         // 登录信息错误
UserAuthenticationException.needApikey(exchange)         // 需要 API Key
UserAuthenticationException.verificationCodeError(exchange) // 验证码错误
```

**使用示例：**

```java title="AuthService.java"
public Mono<UserDTO> authenticate(ServerWebExchange exchange) {
    String token = exchange.getRequest().getHeaders().getFirst("Authorization");
    if (token == null) {
        // [!code highlight:2]
        // 使用工厂方法创建认证异常
        throw UserAuthenticationException.userNotLogin(exchange);
    }
    return tokenService.validate(token)
            .switchIfEmpty(Mono.error(
                    UserAuthenticationException.tokenExpired(exchange)
            ));
}
```
