# ResultUtil

`ResultUtil` 位于 `com.xlf.utility.webflux`。  
它在 Reactor Context 中读取 `contextId` 与 `duration`，并返回响应式 `Mono<ResponseEntity<...>>`。

## 方法签名

```java title="ResultUtil.java"
public static Mono<ResponseEntity<BaseResponse<Void>>> success(String message)

public static <T> Mono<ResponseEntity<BaseResponse<T>>> success(String message, T data)

public static <T> Mono<ResponseEntity<BaseResponse<T>>> error(
        ErrorCode errorCode, String errorMessage, T data)
```

## 与 MVC 版本差异

- MVC：返回 `ResponseEntity<BaseResponse<T>>`（同步）  
- WebFlux：返回 `Mono<ResponseEntity<BaseResponse<T>>>`（响应式）

## 使用示例

```java title="UserController.java"
@GetMapping("/{id}")
public Mono<ResponseEntity<BaseResponse<UserDTO>>> getUser(@PathVariable Long id) {
    return userService.getUserById(id)
            .flatMap(user -> ResultUtil.success("获取成功", user))
            .switchIfEmpty(ResultUtil.error(ErrorCode.RESOURCE_NOT_FOUND, "用户不存在", null));
}
```

## 注意事项

- `success` 系列方法中的 `code` 使用 `200`，不是 `ErrorCode` 的业务码。  
- `error` 推荐在全局异常处理器中使用。  
- 若上下文未注入，`context` 与 `duration` 可能为 `null`。
