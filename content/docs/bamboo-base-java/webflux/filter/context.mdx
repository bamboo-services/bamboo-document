# ContextFilter

`ContextFilter` 是 `bamboo-webflux` 模块最核心的过滤器，负责在每个请求的 Reactor Context 中注入链路追踪标识（UUID）与请求开始时间戳。这些信息将被 `ContextHolder`、`ResultUtil` 和日志切面等组件消费。

## 类定义

```java title="ContextFilter.java"
// [!code highlight:4]
// 最高优先级的上下文注入过滤器
@Order(Ordered.HIGHEST_PRECEDENCE)
public class ContextFilter implements WebFilter {

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, WebFilterChain chain) {
        // ...
    }
}
```

## 与 MVC 版本的本质区别

MVC 版本的 `ContextFilter` 使用 `ThreadLocal` 存储上下文信息，而 WebFlux 版本使用 Reactor Context：

```java title="对比"
// MVC 版本 - ThreadLocal
public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) {
    ThreadLocal<String> contextId = new ThreadLocal<>();
    contextId.set(UUID.randomUUID().toString());
    chain.doFilter(request, response);
    contextId.remove();
}

// [!code highlight:6]
// WebFlux 版本 - Reactor Context
public Mono<Void> filter(ServerWebExchange exchange, WebFilterChain chain) {
    String uuid = UUID.randomUUID().toString();
    return chain.filter(exchange)
            .contextWrite(Context.of(
                    ContextConstant.CONTEXT_KEY, uuid,
                    ContextConstant.START_TIME_KEY, System.currentTimeMillis()
            ));
}
```

## 工作流程

```
请求进入
  │
  ├── 1. 检查请求方法是否为 OPTIONS → 跳过
  │
  ├── 2. 检查请求路径是否在排除列表中 → 跳过
  │
  ├── 3. 生成 UUID（或从请求头复用）
  │
  ├── 4. 设置 MDC 日志上下文
  │
  ├── 5. 通过 contextWrite 注入 Reactor Context
  │     ├── ContextConstant.CONTEXT_KEY → UUID
  │     └── ContextConstant.START_TIME_KEY → 当前时间戳
  │
  └── 6. 传递至下一个过滤器
```

## 上下文复用

当 `enableInput` 配置为 `true` 时，`ContextFilter` 会检查请求头中是否携带了外部传入的链路追踪 UUID。若存在则复用该 UUID，而非重新生成。这在微服务间调用时可保持链路追踪的连贯性。

```java title="ContextFilter.java"
// [!code highlight:5]
// 检查请求头中是否携带外部 UUID
String externalContextId = exchange.getRequest().getHeaders()
        .getFirst(ContextConstant.CONTEXT_KEY);
String uuid = (enableInput && externalContextId != null)
        ? externalContextId
        : UUID.randomUUID().toString();
```

## 排除路径

通过 `ContextWebFluxProperties.excludeUrls` 配置的路径将被跳过，不注入上下文信息：

```yaml title="application.yml"
utility:
  context:
    webflux:
      exclude-urls:
        - /actuator/**
        - /health
        - /favicon.ico
```

## MDC 集成

`ContextFilter` 在注入 Reactor Context 的同时，还会设置 SLF4J MDC（Mapped Diagnostic Context），使得日志框架能够在日志输出中包含链路追踪信息：

```java title="ContextFilter.java"
// [!code highlight:2]
// 设置 MDC，使日志输出包含 contextId
MDC.put("contextId", uuid);
```

在 `logback.xml` 中可通过 `%X{contextId}` 引用：

```xml title="logback.xml"
<pattern>%d{yyyy-MM-dd HH:mm:ss} [%X{contextId}] %-5level %logger{36} - %msg%n</pattern>
```

## 配置属性

`ContextFilter` 的行为由 `ContextWebFluxProperties` 控制：

<TypeTable
  type={{
    enableInput: {
      description: '是否允许外部传入链路追踪 UUID。',
      type: 'boolean',
      default: 'true',
    },
    enableOutput: {
      description: '是否在响应头中输出链路追踪 UUID。',
      type: 'boolean',
      default: 'true',

    gatewayMode: {
      description: '是否启用网关模式。',
      type: 'boolean',
      default: 'false',
    },
    excludeUrls: {
      description: '排除的 URL 路径列表，支持 Ant 风格匹配。',
      type: 'List<String>',
      default: '空列表',
    },
  }}
/>

## 自动注册

`ContextFilter` 由 `WebFluxSdkAutoConfiguration` 自动注册：

```java title="WebFluxSdkAutoConfiguration.java"
@Bean
public ContextFilter contextFilter(ContextWebFluxProperties properties) {
    return new ContextFilter(properties);
}
```
