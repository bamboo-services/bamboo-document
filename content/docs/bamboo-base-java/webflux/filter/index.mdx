# WebFilter 概览

`bamboo-webflux` 提供了一组基于 Spring WebFlux `WebFilter` 接口的过滤器，用于在请求处理管道中执行横切关注点逻辑，包括上下文注入、CORS 校验、OPTIONS 预检处理和权限校验。

## WebFilter 与 Servlet Filter 的区别

在 Spring WebFlux 中，过滤器不再基于 Servlet 规范的 `javax.servlet.Filter`，而是使用响应式的 `WebFilter` 接口：

```java title="WebFilter.java"
public interface WebFilter {
    // [!code highlight:3]
    // WebFilter 接口方法，返回 Mono<Void> 表示异步完成
    Mono<Void> filter(ServerWebExchange exchange, WebFilterChain chain);
}
```

| 对比项 | Servlet Filter | WebFilter |
|--------|---------------|-----------|
| 线程模型 | 阻塞式，一请求一线程 | 非阻塞式，事件循环 |
| 请求对象 | `HttpServletRequest` | `ServerWebExchange` |
| 返回类型 | `void` | `Mono<Void>` |
| 链路调用 | `FilterChain.doFilter()` | `WebFilterChain.filter()` |

## 过滤器列表

| 过滤器 | Order | 功能 |
|--------|-------|------|
| `ContextFilter` | `HIGHEST_PRECEDENCE` | 注入链路追踪 UUID 与开始时间至 Reactor Context |
| `AllowCorsWebFilter` | `HIGHEST_PRECEDENCE` | 校验请求来源域名并设置 CORS 响应头 |
| `AllowOptionWebFilter` | `HIGHEST_PRECEDENCE + 1` | 处理 OPTIONS 预检请求，直接返回 200 |
| `PermissionWebFilter` | 默认 | 抽象过滤器，校验 `@NeedPermission` 注解 |

## 执行顺序

过滤器按照 `@Order` 注解的值从小到大依次执行：

```
请求进入
  │
  ├── ContextFilter (HIGHEST_PRECEDENCE)
  │     └── 注入 UUID、开始时间至 Reactor Context
  │
  ├── AllowCorsWebFilter (HIGHEST_PRECEDENCE)
  │     └── 校验域名、设置 CORS 头
  │
  ├── AllowOptionWebFilter (HIGHEST_PRECEDENCE + 1)
  │     └── OPTIONS 请求直接返回 200
  │
  ├── PermissionWebFilter (用户实现)
  │     └── 校验接口权限
  │
  └── Controller 处理
```

## 与 MVC 过滤器的对照

| MVC 过滤器 | WebFlux 过滤器 |
|-----------|---------------|
| `ContextFilter` (implements Filter) | `ContextFilter` (implements WebFilter) |
| `AllowCorsFilter` | `AllowCorsWebFilter` |
| `AllowOptionFilter` | `AllowOptionWebFilter` |
| `PermissionFilter` (abstract) | `PermissionWebFilter` (abstract) |

两者在功能上完全对等，仅在接口实现与请求对象类型上有所不同。
