# ResultUtil

`ResultUtil` 位于 `com.xlf.utility.mvc`，用于统一构建 MVC 响应。  
它会从 `ContextHolder` 读取 `contextId` 与 `duration`，并写入 `BaseResponse`。

## 方法签名

```java title="ResultUtil.java"
public static ResponseEntity<BaseResponse<Void>> success(String message)

public static <T> ResponseEntity<BaseResponse<T>> success(String message, T data)

public static <T> ResponseEntity<BaseResponse<T>> error(
        ErrorCode errorCode, String errorMessage, T data)
```

## 使用示例

```java title="UserController.java"
@GetMapping("/{id}")
public ResponseEntity<BaseResponse<UserVO>> getUser(@PathVariable Long id) {
    UserVO user = userService.getUserById(id);
    return ResultUtil.success("查询成功", user);
}
```

```java title="GlobalExceptionHandler.java"
@ExceptionHandler(BusinessException.class)
public ResponseEntity<BaseResponse<Object>> handleBusiness(BusinessException e) {
    return ResultUtil.error(e.getErrorCode(), e.getErrorMessage(), e.getData());
}
```

## 注意事项

- `success` 系列方法中的 `code` 使用 `200`，不是 `ErrorCode` 中的业务码。  
- `error` 推荐在全局异常处理器中调用，业务层优先抛异常而不是直接返回错误响应。  
- 若请求未经过 `ContextFilter`，`context` 与 `duration` 可能为 `null`。
