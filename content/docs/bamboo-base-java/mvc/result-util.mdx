# ResultUtil

`ResultUtil` 是一个静态工具类，用于构建符合 `BaseResponse<T>` 规范的标准化 HTTP 响应。该工具类会自动从 `ContextHolder` 中获取上下文标识符（Context ID）和请求耗时（Duration），确保每个响应均携带完整的追踪信息。

## 方法签名

### success

构建成功响应，HTTP 状态码固定为 `200 OK`。

```java title="ResultUtil.java"
// [!code highlight:2]
// 不携带数据的成功响应
public static ResponseEntity<BaseResponse<Void>> success(String message)

// [!code highlight:2]
// 携带数据的成功响应
public static <T> ResponseEntity<BaseResponse<T>> success(String message, T data)
```

### error

构建错误响应，HTTP 状态码根据 `ErrorCode` 自动计算。

```java title="ResultUtil.java"
// [!code highlight:2]
// HTTP 状态码 = errorCode.getCode() / 100
public static <T> ResponseEntity<BaseResponse<T>> error(ErrorCode errorCode, String errorMessage, T data)
```

## 状态码映射

`ResultUtil` 的 HTTP 状态码由 `ErrorCode` 的编码值决定：

| ErrorCode 编码 | HTTP 状态码 | 含义 |
|---------------|------------|------|
| 20000 | 200 | 成功 |
| 40000 | 400 | 请求参数错误 |
| 40100 | 401 | 未授权 |
| 40300 | 403 | 权限不足 |
| 40400 | 404 | 资源不存在 |
| 50000 | 500 | 服务器内部错误 |

## 使用示例

### 成功响应（无数据）

```java title="UserController.java"
@PostMapping("/register")
public ResponseEntity<BaseResponse<Void>> register(@RequestBody RegisterDTO dto) {
    userService.register(dto);
    // [!code highlight:2]
    // 返回成功响应，自动注入 context 和 duration
    return ResultUtil.success("注册成功");
}
```

### 成功响应（携带数据）

```java title="UserController.java"
@GetMapping("/{id}")
public ResponseEntity<BaseResponse<UserVO>> getUser(@PathVariable Long id) {
    UserVO user = userService.getUserById(id);
    // [!code highlight]
    return ResultUtil.success("查询成功", user);
}
```

### 错误响应

```java title="UserController.java"
@GetMapping("/validate")
public ResponseEntity<BaseResponse<Void>> validate(@RequestParam String token) {
    if (!tokenService.isValid(token)) {
        // [!code highlight:2]
        // HTTP 状态码将被设置为 401 (40100 / 100)
        return ResultUtil.error(ErrorCode.UNAUTHORIZED, "令牌已过期", null);
    }
    return ResultUtil.success("验证通过");
}
```

### 在异常处理器中使用

```java title="GlobalExceptionHandler.java"
@ExceptionHandler(BusinessException.class)
public ResponseEntity<BaseResponse<Void>> handleBusinessException(BusinessException e) {
    // [!code highlight]
    return ResultUtil.error(e.getErrorCode(), e.getMessage(), null);
}
```

## 自动上下文注入

`ResultUtil` 内部通过 `ContextHolder` 获取以下信息：

1. **Context ID** — 调用 `ContextHolder.getContextId()` 获取当前请求的唯一标识
2. **Duration** — 调用 `ContextHolder.getDuration()` 计算从请求开始到响应构建的耗时

因此，在使用 `ResultUtil` 之前必须确保 `ContextFilter` 已完成上下文初始化。若上下文未初始化，Context ID 和 Duration 将使用默认值。
